# Market-Brief Banner — Design Spec

**Date:** 2026-05-05
**Status:** Approved (brainstorming complete; awaiting implementation plan)
**Branch context:** `feat/market-brief` (off `feat/recommendations`, which is already an open PR)

## Goal

Add a single-line market-tone banner at the top of `/recommendations` that summarizes the overall tape in one short headline plus 1–3 driver phrases and a bias chip (BULLISH / BEARISH / NEUTRAL). The banner is generated by a small Haiku call from market-wide news + SPY/QQQ/IWM quotes and refreshes alongside the existing per-ticker recommendation flow.

## Scope

### In scope

- New backend `generate_market_brief()` in `recommendation_service.py` that pulls market-wide news + index quotes and calls Haiku once for a structured headline.
- A new prompt block in `recommendation_prompt.py` (`MARKET_BRIEF_SYSTEM_PROMPT` + `build_market_brief_user_message`).
- Bundled execution: `generate_all` runs `generate_market_brief` first, emits a `recommendation.market_brief` WS event, and includes `market_brief` in the persisted `RecommendationRun.payload_json`.
- `/recommendations/latest` returns `market_brief` alongside `cards` and `sources`.
- New frontend type `MarketBrief` and a new `<MarketBriefBanner />` component rendered above `<SourcesStrip />` in `RecommendationsView`.
- WS hook handles the new event.

### Out of scope (deliberate, YAGNI)

- No separate "/market" route; this is a banner only.
- No sector-ETF heatmap, top-movers list, or macro narrative paragraph.
- No standalone refresh button for the banner — it refreshes when the user clicks Generate (matching the existing model).
- No new agent tool; nothing about this changes the trade flow on `/`.
- No persistence beyond the existing `RecommendationRun` table.

## Decisions captured during brainstorming

| # | Question | Choice | Rationale |
|---|---|---|---|
| Q1 | Banner refresh model | Bundled into `generate_all` (refreshes on Generate) | One mental model — Generate refreshes both banner and cards atomically. No new infra. |
| Q2 | Inputs to the brief | `general_news` + `SPY/QQQ/IWM` mid prices | Already-built primitives; covers the broad tape (large-cap, tech-heavy, small-cap). |
| Q3 | Persistence | Embedded in existing `RecommendationRun.payload_json` | One row per run keeps the cache simple; no new table. |
| Q4 | Streaming order | Emit `recommendation.market_brief` AFTER `recommendation.discovery`, BEFORE first card | Banner appears as soon as it's ready; cards continue to stream in below. |
| Q5 | Failure behavior | Omit the banner and emit `recommendation.market_brief_warning` | One brief failure must not block the per-ticker run. |

## Architecture

### Backend (`backend/app/`)

**Extended — `app/agent/recommendation_prompt.py`**

Add two new symbols (alongside the existing per-ticker `SYSTEM_PROMPT` / `build_user_message`):

- `MARKET_BRIEF_SYSTEM_PROMPT` — strict JSON-only output. Schema:
  ```
  {
    "bias": "bullish" | "bearish" | "neutral",
    "headline": string ≤ 100 chars,
    "drivers": array of 1..3 strings, each ≤ 60 chars
  }
  ```
- `build_market_brief_user_message(index_quotes: dict[str, float | None], news_items: list[dict]) -> str` — formats the quotes (SPY/QQQ/IWM) and up to 8 market-wide headlines + summaries into a user message.

Reuses the existing `build_strict_retry_message()` for retry on malformed output.

**Extended — `app/services/recommendation_service.py`**

- `async generate_market_brief() -> dict | None` — orchestrator-internal helper:
  1. `quotes = alpaca_service.get_latest_prices(["SPY", "QQQ", "IWM"])` (off-loop via `run_in_executor`).
  2. `news = news_service.get_general_news()` (off-loop), best-effort — failure → `None` for that input.
  3. If both quotes are all-`None` AND news is empty → return `None` (nothing to summarize).
  4. Call Haiku once with the new prompt; parse strict JSON via a small dedicated parser (`_parse_brief_json`) that validates `bias`, `headline ≤ 100`, `drivers` list of 1–3 strings each ≤ 60.
  5. Retry once on malformed (using `build_strict_retry_message`); on persistent malformed, return `None`.
  6. Return `{bias, headline, drivers, updated_at}` (ISO timestamp).

- Modify `generate_all(emit)`:
  - Right after the existing `recommendation.discovery` emit (and any `discovery_warning`), call `await generate_market_brief()`.
  - If non-None: emit `{"type": "recommendation.market_brief", "data": brief}`.
  - If None: emit `{"type": "recommendation.market_brief_warning", "data": {"message": "market_brief_unavailable"}}`.
  - Add `market_brief` to the persisted `payload` (alongside `cards`, `sources`, `run_id`, `generated_at`). The value is the brief dict or `null`.

- `get_latest_run()` continues to return `json.loads(row.payload_json)`. The new `market_brief` field flows through automatically.

**Extended — `backend/app/api/http.py`**

Endpoint shape adjustment for `GET /recommendations/latest`: when no run exists, the empty-shape response gains a `market_brief: null` key so clients always see the field.

```python
return {
    "run_id": None,
    "generated_at": None,
    "cards": [],
    "sources": {"watchlist": [], "positions": [], "discover": []},
    "market_brief": None,
}
```

No change needed when a run exists — the persisted payload now includes `market_brief`.

**Extended — `backend/app/api/ws.py`**

No change required. The existing `emit` helper passes through the new event types via the same code path.

**Backend test additions:**

- `test_generate_market_brief_happy_path` — mocks `_anthropic`, `get_general_news`, `get_latest_prices`; asserts shape.
- `test_generate_market_brief_malformed_returns_none` — mocks Anthropic to return non-JSON twice; asserts `None`.
- `test_generate_market_brief_no_inputs_returns_none` — both inputs empty/None; asserts `None` and no Anthropic call.
- Extend `test_generate_all_streams_and_persists` to assert `market_brief` is present in the persisted payload (mock `generate_market_brief` to return a fixed dict).
- Extend `test_recommendations_latest_empty` to assert `market_brief is None` in the empty shape.

```typescript
| { type: "recommendation.market_brief_warning"; ts?: string; data: { message: string } }
```

**Extended — `lib/recommendations-ws.ts`**

Add `marketBrief: MarketBrief | null` to `RecommendationsState` (initialized to `null`). Handle the two new event types in the switch:

- `recommendation.market_brief` → `setState({...prev, marketBrief: data})`
- `recommendation.market_brief_warning` → leave `marketBrief: null`, optionally set a separate `briefWarning` field (omit from v1; the absence of a banner is the signal).

**New — `components/recommendations/market-brief-banner.tsx`**

```tsx
"use client";
import type { MarketBrief } from "@/types/recommendations";

const BIAS_COLOR: Record<string, string> = {
  bullish: "var(--up)",
  bearish: "var(--down)",
  neutral: "var(--fg-dim)",
};

export function MarketBriefBanner({ brief }: { brief: MarketBrief | null }) {
  if (!brief) return null;
  const color = BIAS_COLOR[brief.bias] ?? "var(--fg-dim)";
  return (
    <section className="px-5 py-4 border-b border-[color:var(--hairline)] flex items-start gap-4">
      <span
        className="font-mono text-[10.5px] tracking-[.22em] px-2 py-[3px] mt-[3px]"
        style={{ background: color, color: "var(--ink)", fontWeight: 700 }}
      >
        {brief.bias.toUpperCase()}
      </span>
      <div className="flex-1">
        <p className="text-[14px] leading-snug">{brief.headline}</p>
        {brief.drivers.length > 0 && (
          <p className="text-[11.5px] font-mono text-[color:var(--fg-mute)] mt-1">
            {brief.drivers.join(" · ")}
          </p>
        )}
      </div>
    </section>
  );
}
```

**Extended — `components/recommendations/recommendations-view.tsx`**

- Read `marketBrief` with the same live/cached priority as the cards: `liveActive ? state.marketBrief : (latest?.market_brief ?? null)`.
- Render `<MarketBriefBanner brief={brief} />` between the warning/error stack and `<SourcesStrip />`.

## Data flow

1. User navigates to `/recommendations`. `RecommendationsView` fetches `/recommendations/latest`. If a prior run exists, `latest.market_brief` is rendered immediately.
2. User clicks **Generate**. WS opens; backend runs `discover_candidates()` → emits `recommendation.discovery` → calls `generate_market_brief()` → emits `recommendation.market_brief` (or `_warning`).
3. Frontend hook updates `state.marketBrief`. Banner appears at the top.
4. Per-ticker generation continues as today; cards stream in.
5. On `recommendation.complete` the orchestrator persists the run with `market_brief` embedded.

## Error handling

- **Quote fetch fails for one or more indices:** other indices proceed; the brief still generates from whatever quotes are available + news.
- **News fetch fails:** brief still generates from quotes alone; the prompt receives `(no recent news available)`.
- **Both quotes all-None and news empty:** brief generation is skipped; `recommendation.market_brief_warning` emitted. No banner.
- **Anthropic returns malformed JSON:** retry once with the strict-retry prompt. If still malformed, return None and emit warning. No banner.
- **Anthropic raises:** caught, return None, emit warning. No banner.
- **Banner failure must not block per-ticker run:** the orchestrator continues regardless of brief outcome.

## Edge cases

- **Cached run from before this feature shipped:** `payload_json` lacks `market_brief`. `get_latest_run` returns the dict as-is; missing field → `None` on the frontend. Banner simply doesn't render. Acceptable.
- **Headline returned as 200 chars:** the parser truncates to ≤ 100 (defensive), preserving the schema.
- **Drivers list returned as `[]` or omitted:** parser accepts `[]` (zero is valid in 0..3); banner renders headline only.
- **Bias not in the enum:** parse fails → retry → if persistent, None.

## Testing

- `test_generate_market_brief_happy_path` — mock `_anthropic` to return canned JSON, mock `get_general_news` and `get_latest_prices`; assert returned dict has `bias`, `headline`, `drivers`, `updated_at` and matches the canned values.

- `test_generate_all_market_brief_warning_when_none` — patch `generate_market_brief` to return None; assert a `recommendation.market_brief_warning` event was emitted and `payload["market_brief"] is None`.
- Update `test_recommendations_latest_empty` to assert `market_brief is None` in the empty shape.
